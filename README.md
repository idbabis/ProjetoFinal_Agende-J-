## **Apresentação**

<br>

O "Agende já" tem como objetivo marcar um horário online para consultas veterinárias, banhos e tosas para o seu pet.
Muitas pessoas ficam descontentes em ligar para as lojas para agendar um horário, muitas vezes quando o cliente necessita de um horário não esta vago. 
Sempre a loja determina o horário e dia que você deve agendar. O intuito do API é facilitar, dar opções, facilidade, ao cliente e não somente aos lojistas, por outro lado os profissionais tem como grande beneficio a organização e a dinâmica com o cliente. 
Tudo fica mais fácil! Descomplicando assim a demora no atendimento de forma geral.

Essa é uma API feita para o projeto final do [{reprograma}](https://reprograma.com.br/), e tem como objetivo agendar um dia e horário especificos para o seu animal, o usuário poderá se cadastrar na plataforma e agendar seu hoário e seu atendimento de preferencia e a loja poderá ver essa agenda com os dias de agendamentos dos usuários.
<br><br>

Um sistema de agendamento online que te permite aceitar reservas e oferecer serviços online, para lojas pets e seus usuários que queiram agendar um horário para seu animalzinho. Facilitando assim a mão de obra física das lojas e usuário e customizando um sistema de agenda. O intuito seria também baixar os custos dos atendimentos e ter preços "acessíveis" ou "populares".

## **Motivações**

<br>

![Capa Para PROJETO FINAL(reprograma) (1)](https://user-images.githubusercontent.com/100974682/181840389-bd2fdaaa-1271-4c41-9557-ae77787e76e8.gif)


<br>

Há 3 anos adotamos a Mel nossa cachorrinha, logo no primeiro dia fomos em 3 lojas pets para ver se havia horário para banho e ligamos para mais duas lojas, porém sem sucesso. Demoramos pelo menos 1hora e 40 minutos nesse looping.
E no final tivemos que dar banho em casa mesmo e ligar outro dia em uma outra loja para agendar um dia e horário.

Se na época tivesse um sistema ou app que fosse de fácil acesso para cadastrar e já tivessemos como agendar na hora, seria muito menos "traumático" e  trabalhoso.  

Essa situação fez a ideia do **Agende Já** surgir, e hoje com o conhecimento em API adquirido no curso pude realizá-lo.

## **Instruções para instalação e contribuições no projeto**

<br>

- Faça um `fork` do projeto através do link [API Aagende Já](https://github.com/idbabis/ProjetoFinal_Agende-Ja.git).

- Copie o código do _fork_ realizado e, no _prompt de comando_ da sua máquina, realize o clone do projeto através do `git clone <link_do_fork_do_repositorio>`;

- Crie uma _branch_ para realizar suas contribuições `git checkout -b feature/<sua_branch>`;

- Instale as dependências necessárias à execução da API através do comando `npm install`;

- No raiz do projeto, renomeie `.env.example` para `.env` e adicione os valores das variáveis `PORT` (porta sugerida `8080`), `MONGODB_URL ou DATABASE_URI` (string de conexão com o banco de dados) e `SECRET` (chave RSA). Elas são necessárias para a execução da API em sua máquina

- Para executar a API, utilize o comando `npm start` no seu terminal;

- Após suas contribuições no projeto, realize o _commit_ com o comando `git commit -m 'sua mensagem'`;

- Para subir o projeto no seu GitHub, basta executar o comando `git push origin feature/<sua_branch>`;

- E finalize criando um novo _Pull Request_ com as contribuições para o projeto original.

## **Features e rotas**

Esta API está sendo escutada na `porta 3000` e para que todas as rotas possam ser acessadas localmente é necessário usar `http://localhost:3000/` ou `https://projetolivre-aula.herokuapp.com/minha-rota-de-documentacao/#/` antes dos endpoints de requisição.

![rodando_na_port3000](https://user-images.githubusercontent.com/100974682/181858130-c3ba683c-a5c0-419a-83f7-7ff8c55b3eec.gif)

### _Manipulando registros como pessoa usuária (requer token de autenticação)_

| Feature                                | Método | Rota                         |
| -------------------------------------- | ------ | ---------------------------- |
| Login                                  | POST   | `/login`                     |
| Cadastro de usuario                    | POST   | `/usuario`                   |
| Todos os usuarios                      | GET    | `/usuarios`                  |
| Todas as lojas                         | GET    | `/lojas`                     |
| Atualizar cadastro                     | PATCH  | `/usuario/:id`               |
| Apagar cadastro                        | DELETE | `/usuario/:id`               |
